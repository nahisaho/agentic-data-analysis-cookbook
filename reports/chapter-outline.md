## 前提

- 対象読者: ARIMデータポータル会員（材料・ナノテク研究者。Python/Jupyter経験あり、MCP未経験）
- 最終ゴール（合格ライン）: 読者が自分の実験データ向けに「データ分析用Skill」を1つ以上自力で作成できること
- 分量目安: 実践書（200〜250ページ規模）
- 期限: 6か月
- ハンズオン標準環境: Python + JupyterLab + Jupyter MCP + GitHub Copilot CLI + ToolUniverse MCP、その他 FastMCP, arXiv MCP, Paper Search MCP など
- 参照:
  - 前回資料（参考・全面刷新）: https://nanonet.go.jp/data_service/data/doc/1778725311_doc_16_0.pdf
  - MCP基準資料: https://qiita.com/hisaho/items/dc64fda8f0db972c392b
  - 装置カテゴリ一覧: https://nanonet.go.jp/data_service/page/textbook.html

## データ型による抽象化（装置カテゴリの網羅方針）

多数の装置カテゴリを個別に深掘りせず、以下の「データ型」に抽象化して共通の分析パターンを提示する。各装置カテゴリはいずれかの型にマッピングする。

| データ型 | 代表装置カテゴリ |
|---|---|
| スペクトル型 | 各種分光（状態分析）、磁気共鳴、質量分析 |
| クロマトグラム・時系列型 | クロマトグラフ、熱分析、プロセス条件ログ |
| 画像・顕微鏡型 | 光学顕微鏡、走査型顕微鏡、透過電子顕微鏡 |
| 回折・散乱パターン型 | 回折・散乱、表面分析 |
| 表形式・プロセス条件型 | 成膜、リソグラフィ、機械/電気/磁気特性 |
| マルチモーダル統合型 | 上記の組み合わせ |

| 章 | タイトル | 責務 | 成果物 |

### 第III部　Skillを作る（目安 50〜60ページ）

| 章 | タイトル | 責務 | 成果物 |
|---|---|---|---|

## 責務分離マップ（重複防止）

| 論点 | 予防・設計側 | 実行・事例側 |
|---|---|---|
| 安全性（データ漏洩・権限） | 第6章（ルール・権限・承認） | 第14章（失敗事例・診断） |
| 評価 | 第7章（設計時の評価仕様） | 第12章（実行後の検証） |
| データ処理 | 第8章（入力のデータ契約・標準化） | 第9章（Skill内部の分析処理） |
| 循環設計問題 | 第7章（設計で回避） | 第14章（事例集） |

## 各ハンズオン章の共通構成

各ハンズオン章（第9・10・11章）は以下を含める。**第9章で丁寧に説明し、第10・11章は差分中心**。共通テンプレート本体は付録Aに集約。

- この章で作るSkillの概要
- 入力仕様 / 出力仕様 / 制約条件
- 評価基準（正確性・再現性・解釈可能性・データ漏洩リスク・レビュー容易性・転用可能性）
- 実行例 / 失敗例 / 改善版
- 他装置カテゴリ（データ型）への転用方法
